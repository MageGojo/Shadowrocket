# 需求说明

## 目标

维护一份可导入 Shadowrocket 的自定义规则配置，在上游广告拦截规则基础上，按个人需求追加或调整分流策略。

## 当前范围

- 下载并保留上游 `sr_ad_only.conf` 原始规则。
- 创建 `custom.conf` 作为后续自定义输出。
- 备份并读取本机 Shadowrocket 的 `sr_ad_only.db` 数据库，便于对照应用内导入后的结构。
- 后续根据明确需求追加规则，例如直连、代理、拒绝、指定代理组等。
- 在上游广告规则基础上，新增 AI / Google / Apple 分流策略组与规则。
- 为 TUN 模式配置 `[General]` 旁路路由（详见技术设计文档）。

## 已确定

- 自定义代理组名称：`OpenAI / Gemini / Claude / GoogleAI / GoogleSearch / Apple / Domestic / Final`。
- 保留完整上游广告 `Reject` 规则，并在其前面追加分流规则（合并方案，而非纯广告）。
- TUN：在配置文件中配置旁路路由，TUN 总开关由 App/VPN 控制。

## 暂未确定

- 是否需要补充流媒体（Netflix / YouTube 等）或其它细分分流。
- 是否需要额外的关键字、IP 段或按应用（App）分流规则。
